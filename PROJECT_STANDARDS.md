# VirtuAI — Project Standards (Top-Tier Creator-Account Bar)

> This document defines the quality floor every published piece of content
> must clear. The bar is set by what works on real AI persona accounts in
> 2026 (andyhan.ai, aitana.lopez, milla.sofia, founder reels at >100K
> followers). Every standard below is enforceable — it maps to a specific
> module or agent in the pipeline. If a piece of output fails any of these
> checks, the Reviewer or Guardian agent should reject and re-run.
>
> The intent is to STOP shipping anything that looks AI-generated and START
> shipping content indistinguishable from a real creator's work.

---

## 1. Visual Standards

### 1.1 Camera & Framing

| Standard | Required | Enforced in |
|---|---|---|
| Camera positioned IN FRONT of subject (not held by subject) | Always | `prompt_writer.py` system prompt |
| No phone visible in subject's hand | Always | `prompt_writer.py` blacklist |
| No "selfie POV" framing | Always | `prompt_writer.py` blacklist |
| Eye-level angle (not below, not bird's-eye) | Always | `prompt_writer.py` |
| Medium shot (head to chest) OR medium close-up (head to upper torso) | Default | `prompt_writer.py` |
| 9:16 aspect ratio for reels | Always | `kling_omni.multi_image_to_video(aspect_ratio="9:16")` |
| Minimum resolution 720×1280 | Always | Kling default |

### 1.2 Environment

| Standard | Required |
|---|---|
| Real-world location: café, office, home workspace, kitchen, gym, terrace, bookstore, hotel lobby, outdoor street | Always |
| Background depth visible (architectural elements, bokeh, windows) | Always |
| Real props in shot (coffee cup, laptop, notebook, plants, books) | Always |
| Background must look INHABITED — not staged | Always |
| ❌ NO studio backgrounds, plain walls, neutral grey | Never |
| ❌ NO held-phone "selfie" framing | Never |

### 1.3 Subject Pose & Behaviour

The subject must be doing ONE of these (auto-rotated per clip):

| Archetype | Description |
|---|---|
| **Café window seat** | Seated at small table, MacBook open at 45°, espresso cup, leaning back slightly, hand on cup or keyboard |
| **Home desk** | Seated at desk, laptop in frame, plant or books on shelf behind, leaning forward when emphasising |
| **Kitchen counter** | Standing, leaning against counter, coffee mug in hand, morning light through window |
| **Walking talk** | Slow walk through office or street, talking while glancing back at camera |
| **Standing office** | Leaning on doorframe or desk in modern office, casual relaxed pose |
| **Outdoor terrace** | Seated on rooftop / café terrace, soft side light, urban background |

NOT acceptable:
- ❌ Subject holding their own phone
- ❌ Stiff straight-on pose with no body language
- ❌ Posed studio "headshot" feel

### 1.4 Lighting

| Acceptable | Description |
|---|---|
| Golden hour | Late afternoon warm side light |
| Soft window light | Diffused daylight from a real window |
| Soft overcast | Even neutral daylight |
| Warm practical | Indoor evening with visible lamps |
| ❌ Studio strobes | Never |
| ❌ Flat fluorescent | Never |

### 1.5 Technical Polish

| Standard | Required |
|---|---|
| Shallow depth of field (background blurred) | Always |
| Film-like grain (subtle) | Always |
| Cinematic colour grade (warm, slight teal in shadows) | Always |
| Photorealistic, not stylised | Always |
| Subtle handheld drift OR static tripod (pick one per clip) | Always |

### 1.6 Identity Consistency

| Standard | Required | Enforced in |
|---|---|---|
| ArcFace identity match ≥ 0.70 vs. canonical Daniel | Per clip | `virtuai/persona/eval/` post-generation check |
| 3-4 reference images passed to Kling multi-image2video | Always | `kling_omni.multi_image_to_video(image_paths=[...])` |
| Reference images from `face_dataset/` (1024×1024 real Daniel), not LoRA outputs | Always | Hard-coded list of canonical refs |
| Daniel's distinctive features preserved: curly textured hair, full beard, defined jawline, olive skin | Always | Manual review + ArcFace gate |

## 2. Audio Standards

### 2.1 Voice

| Standard | Required | Enforced in |
|---|---|---|
| Daniel's native voice generated by the video model (Kling 3.0; Seedance 2.0 fallback), never raw synthetic TTS | Always | KIE.ai render (`sound=true`) |
| Clear, intelligible speech — no robotic or muffled artefacts | Always | Native model audio |
| Clean — no clipping, no clicks, no breathiness artifacts | Per clip | Whisper transcription QA |
| Pacing: 140-160 wpm (natural conversational) | Per clip | Script-driven |
| Spoken line embedded in each shot prompt (`Says: "..."`) | Always | Reel script |

### 2.2 Lip Sync

| Standard | Required | Enforced in |
|---|---|---|
| Lip movement matches the spoken audio | Per clip | Native to the video model (Kling 3.0 / Seedance 2.0) |
| Lip-sync is native to the model — no separate lip-sync pass | Always | Single render step |
| ❌ NO un-synced "talking shape" mouth motion | Never | Native audio render |

### 2.3 Background Audio

| Standard | Required |
|---|---|
| Background music bed at -22 dB (subtle, never distracting) | Optional but recommended |
| Music: lifestyle / lo-fi / ambient (no vocals, no copyrighted tracks) | Always |
| Sound effects on cuts (subtle whoosh, ~-18 dB) | Optional |
| Source: YouTube Audio Library, Freesound, ZapSplat (all free) | Always |

## 3. Caption Standards

Captions are the **highest-leverage visual win**. They are the difference between "AI demo" and "actual viral content."

### 3.1 Style

| Standard | Required |
|---|---|
| Word-by-word reveal (CapCut style), NOT full sentence at once | Always |
| 1-3 words per card | Always |
| Centered in lower-middle third of frame (Y ≈ 65-75% from top) | Always |
| Bold sans-serif font: Anton, Montserrat Black, or Inter ExtraBold | Always |
| Font size: ~10% of video height (large, dominant) | Always |
| White text by default | Always |
| Yellow / lime / red highlight on the 1-2 most important words per sentence | Per sentence |
| Black drop shadow + 4-6px outline for legibility on any background | Always |
| Slight pop animation per word (scale 0.8 → 1.0, 100ms) | Always |
| Burned into the video, never as a separate srt | Always |

### 3.2 Timing

| Standard | Required | Enforced in |
|---|---|---|
| Word timestamps from Whisper (word-level) | Always | `caption_generator.py` (to be built) |
| Each word card displayed for its actual spoken duration | Always | ASS subtitle format with PerWord timing |
| Captions never lag behind audio by >50 ms | Always | Whisper sync |

## 4. Hook Standards (First 3 Seconds)

The opening 3 seconds determine whether viewers stay or scroll. Treated as a separate creative unit.

### 4.1 Visual Hook

| Standard | Required |
|---|---|
| Text overlay in first 0-3 seconds reinforcing the verbal hook | Always |
| Hook text style: larger than captions, top-third of frame, single sentence | Always |
| Animated entry (typewriter, fade, or scale-up in first 200 ms) | Always |
| Hook stays on screen for 3 seconds, then fades | Always |

### 4.2 Verbal Hook

The first sentence Daniel speaks must follow one of these proven viral patterns:

| Pattern | Example |
|---|---|
| Contrarian | "Stop trying to scale. Start systemizing." |
| Specific claim | "I built an AI team for $40/month. It replaced 3 freelancers." |
| Bold prediction | "Junior marketers won't exist in 18 months." |
| Counter-intuition | "Most founders confuse motion with progress." |
| Storytime opener | "I asked Claude to audit my business. The first answer was brutal." |
| Personal stat | "This one change 10x'd my output last quarter." |

NOT acceptable as opener:
- ❌ "Hey everyone, today I'm going to talk about..."
- ❌ "So a lot of people ask me about..."
- ❌ Any generic introduction

## 5. Content Standards

### 5.1 Reel Structure (15-30 second optimal)

Every reel must hit these beats in order:

| Beat | Duration | Purpose |
|---|---|---|
| **Hook** | 0-3s | Stop the scroll |
| **Problem / Setup** | 3-7s | Frame why this matters |
| **Insight** | 7-18s | The actual value — specific, named |
| **Proof / Specifics** | 18-25s | Numbers, examples, evidence |
| **CTA** | 25-30s | Follow / save / comment prompt |

### 5.2 Specificity

| Standard | Required |
|---|---|
| Specific numbers (not "a lot", say "11 hours/week") | Always |
| Named tools, named systems, named outcomes | Always |
| Personal experience markers ("I did X", "my team", "last month") | Always |
| ❌ NO generic motivational language | Never |
| ❌ NO "hustle / grind" tropes | Never |

### 5.3 Topic Novelty (Anti-Repetition)

| Standard | Required | Enforced in |
|---|---|---|
| Every reel topic checked against FAISS topic-memory index | Always | `virtuai/persona/topic_memory.py` |
| Cosine similarity ≥ 0.85 with existing topic → REJECT | Always | Existing threshold |
| Topic embedding added to index AFTER successful publish | Always | Existing system |

## 6. Pre-Publish QA Checklist

Every reel must pass ALL of these before the Publisher Agent ships it. Enforced by an upgraded `Reviewer Agent` checklist:

```
[ ] Visual: camera positioned in front of subject (no selfie POV)
[ ] Visual: no phone visible in subject's hand
[ ] Visual: real-world environment with depth and props
[ ] Visual: eye-level medium shot, shallow DOF
[ ] Visual: subject matches Daniel (ArcFace ≥ 0.70)
[ ] Visual: 9:16 aspect ratio
[ ] Audio: native model voice (Kling 3.0 / Seedance 2.0)
[ ] Audio: native lip-sync from the video model (single render step)
[ ] Audio: background music at -22 dB (if applicable)
[ ] Captions: word-by-word, large, centered, white + colored highlights
[ ] Captions: synced via Whisper word-level timestamps
[ ] Hook: text overlay in first 3 seconds
[ ] Hook: verbal opener matches viral pattern (not "hey everyone...")
[ ] Content: 15-30 second duration
[ ] Content: hits all 5 beats (hook / problem / insight / proof / CTA)
[ ] Content: specific numbers, named tools, personal experience
[ ] Content: topic passes novelty check (FAISS < 0.85 cosine match)
[ ] Caption: platform-specific copy generated (LinkedIn ≠ Instagram)
[ ] Hashtags: 5-8 hashtags, mix of niche + broad
[ ] Routing: reel → Instagram + YouTube Shorts; images → Instagram
```

If any check fails: Reviewer Agent flags for revision; Guardian Agent prevents publish.
